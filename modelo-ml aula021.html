Arquitetura de Machine Learning

Sistema de Recomendação por Perfil de Aluno

Análise técnica para Graduação em Computação

O Problema de Negócio

O código implementa um sistema de recomendação educacional que personaliza o ensino baseando-se em duas dimensões:

  • Como o aluno aprende (Formato do conteúdo).
  • O que o aluno deve estudar (Tópico sugerido).

Solução: Uma arquitetura composta (Ensemble/Híbrida) usando dois algoritmos distintos.

Engenharia de Dados (Input)

A matriz X contém 6 características (features) por aluno:

Perfil de Aprendizagem Desempenho (Notas) 

# Dados: [Visual, Auditivo, Prático | Nota_Py, Nota_Design, Nota_Agile]
X = np.array([
    [1,0,0,  5,1,1],  # Aluno 1: Visual, bom em Python
    [0,1,0,  1,5,2],  # Aluno 2: Auditivo, bom em Design
    ...
])

Observem que os dados são heterogêneos: 3 colunas categóricas (One-Hot) e 3 colunas numéricas contínuas.

Estratégia de "Slicing"

O sistema divide o input para alimentar modelos especializados.

Modelo A (Estilo)

X[:, :3]
Usa apenas: Visual, Auditivo, Prático.
➔

Modelo B (Conteúdo)

X[:, 3:]
Usa apenas: Notas de Python, Design, Agile.

Modelo 1: Decision Tree Classifier

tree = DecisionTreeClassifier(max_depth=3).fit(X[:, :3], y_format)
  • Objetivo: Prever o Formato (Vídeo, Podcast, Exercício).
  • Por que Árvore? O perfil de aprendizagem (Visual/Auditivo) segue regras lógicas claras ("Se Visual, então Vídeo").
  • Hiperparâmetro: max_depth=3.
    • Limita a profundidade da árvore.
    • Evita Overfitting (decorar os dados) e melhora a generalização.

Modelo 2: K-Nearest Neighbors (KNN)

knn = KNeighborsClassifier(n_neighbors=1).fit(X[:, 3:], y_topic)
  • Objetivo: Recomendar o Tópico de estudo.
  • Lógica: Baseada em similaridade vetorial.
  • Funcionamento:
    1. O modelo plota as notas do aluno num espaço 3D.
    2. Encontra o aluno mais próximo (vizinho) na base histórica.
    3. Recomenda o tópico que foi útil para aquele perfil similar.

Persistência do Modelo (Pickle)


final_model = {"tree": tree, "knn": knn}
        
with open("ia_models.pkl", "wb") as f:
    pickle.dump(final_model, f)
  • Serialização: O objeto Python (dicionário contendo os dois modelos treinados) é convertido em um fluxo de bytes.
  • Arquivo .pkl: Permite que este modelo seja carregado posteriormente por uma API (Flask/FastAPI) sem necessidade de retreinar.
  • Empacotamento: Note que salvamos um dicionário, encapsulando toda a inteligência em um único arquivo.

Resumo da Arquitetura

Pontos Chave para Levar

  1. O pré-processamento (fatiamento de arrays) é tão importante quanto o modelo.
  2. Não existe "bala de prata": usamos Árvores para regras lógicas e KNN para similaridade numérica.
  3. MLOps e Persistência são fundamentais para transformar código em produto.

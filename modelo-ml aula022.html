Foco: Lógica de Separação e Espaços Vetoriais

1. Contexto dos Dados

Antes de entender o modelo, precisamos visualizar o espaço de características (Feature Space).


# Feature Slicing
X_style = X[:, :3]  # [Visual, Auditivo, Prático] (Categórico/Binário)
X_grades = X[:, 3:] # [Nota_Py, Nota_Design, Nota_Agile] (Contínuo)

Por que separar?

  • X_style representa estados discretos (0 ou 1). Ideal para regras condicionais.
  • X_grades representa magnitudes numéricas. Ideal para cálculo de distância.

2. Decision Tree Classifier

O algoritmo cria um fluxograma hierárquico. Seu objetivo é reduzir a Entropia (desordem) dos dados a cada "pergunta" (split).

Como o algoritmo "pensa":

  1. Ele analisa todas as 3 colunas (Visual, Auditivo, Prático).
  2. Calcula o ganho de informação: "Qual pergunta divide melhor os formatos de conteúdo?"
  3. Exemplo: Se eu separar quem é "Visual", eu iso todos que precisam de "Vídeo"?

Visualização da Árvore (Logic Flow)

Com max_depth=3, a árvore toma decisões sequenciais.

Interpretação: A árvore aprendeu que a característica "Visual" é a mais discriminante para definir o formato vídeo.

3. K-Nearest Neighbors (KNN)

Diferente da árvore, o KNN não cria regras. Ele memoriza a posição dos dados no espaço vetorial.

Métrica de Distância (Euclidiana):

Para um novo aluno $A$ e um aluno existente $B$, calculamos a semelhança baseada nas notas:

\[ d(A, B) = \sqrt{(Py_A - Py_B)^2 + (Des_A - Des_B)^2} \]

n_neighbors=1 significa que o algoritmo é "guloso": ele confia cegamente no vizinho mais próximo, ignorando a média dos demais.

Comparativo de Aprendizagem

Característica Decision Tree KNN
Tipo de Modelo Baseado em Regras (Eager Learning) Baseado em Instância (Lazy Learning)
Fronteira de Decisão Ortogonal (retângulos no espaço) Complexa (Voronoi tessellation)
Interpretabilidade Alta (pode ser desenhada) Média (justificativa por similaridade)
No nosso código Define o Formato (Lógica Rígida) Define o Tópico (Similaridade Fluida)

O artefato final (.pkl)

Ao salvar o dicionário, estamos serializando duas lógicas distintas:


final_model = {
    "tree": tree, # Salva as regras if-else aprendidas
    "knn": knn    # Salva os pontos (X_grades) para comparação futura
}

⚠️ Atenção: O KNN é "pesado" para salvar se o dataset for gigante, pois ele precisa guardar os dados de treino para calcular distâncias no futuro. A Árvore é leve, pois guarda apenas as regras.

Conclusão

  • Usamos Árvores para dados categóricos onde o padrão é claro ("Visual prefere Vídeo").
  • Usamos KNN para dados numéricos onde queremos encontrar "perfis gêmeos" baseados em notas.
  • O sucesso da IA depende de escolher o algoritmo certo para a natureza estatística dos dados.
